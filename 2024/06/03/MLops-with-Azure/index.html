End-to-end MLOps with Azure (1)

Posted on 2024-06-03 Edited on 2024-06-07 In Machine Learning Views: Word count in article: 711 Reading time ≈ 3 mins.

I am writing a series of blogs aiming to address argubally the most significant divides between Machine Learning (ML) practitioners in academia and industry: the disparity between development and deployment.

This blog talks about some basic concepts in MLOps, and the very first step to get a Azure ML job up and running. I organize reliable contents (from the references at the end) and add my inputs.

Typically, researchers, even those focused on applications, conclude their workflow once the model is evaluated (and hopefully generated strong performance). However, to deploy the developed model to real-world problems, the crucial missing step here is to “operationalize” the developed model.

The key steps to operationalize a ML model:

  • Convert the model training to a robust and reproducible pipeline.
  • Test the code and the model in a development environment.
  • Deploy the model in a production environment.
  • Automate the end-to-end process.

Here we follow the learning process provided by Microsoft (therefore we are using Azure). I will also add my thoughts to make our learning more efficient and focused.

The Overall Architecture

The overall architecture of developing and deployment a ML model with Azure contains 4 steps:

  1. All data will be stored in an Azure Blob Storage, which will be managed by the data engineer.
  2. The infrastructure team will create necessary Azure resources like the Azure Machine Learning workspace.
  3. The data scientist will focus on the inner loop: developing and training the model.
  4. The machine learning engineer will take the trained model and deploy it in the outer loop.

The main goal: create a robust and reproducible solution with the output from data scientists.

To define a job in Azure Machine Learning, you can create a YAML file. Whether you want to run one script as a command job or multiple scripts sequentially as a pipeline. For both command and pipeline jobs, you’ll need to create a YAML file, which details:

  • Which scripts to run.
  • What the inputs and outputs are for each script.
  • The compute that will be used to run the scripts.
  • The environment that needs to be installed on the compute to run the scripts.

CLI v2 can be used to run an Azure ML job. It can be installed on the local device or be used with Azure Cloud Shell. In both ways you need to install the Azure ML extension. Installing on windows can be done by:

az extension add -n ml -y

Then, with the access to the Azure subscription ready, you can submit the job with:

az ml job create --file job.yml

For YAML commands, see YAML commands for AzureML.
